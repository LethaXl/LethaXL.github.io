How I Built This Page with Materialize

For this project, I used Materialize, a CSS framework that helps design nice, responsive websites. Here's how I installed the framework and built the page:

1. Installing Materialize

To install Materialize, I didn’t have to download anything to my computer. Instead, I used a special link called a CDN (Content Delivery Network). A CDN is a shortcut to get the Materialize framework directly from the internet, without having to store it on my own computer. I added this link in the <head> section of my HTML page:

<link href="https://cdn.jsdelivr.net/npm/materialize-css@1.0.0/dist/css/materialize.min.css" rel="stylesheet">

This allowed me to use Materialize’s styles on my page.

2. Building the Page

Once I had Materialize installed, I started creating my page. I used Materialize's grid system to arrange everything on the page, so it would look good on both small and large screens. For example:

  • A header at the top with the title "How I Built This Page with Materialize".
  • Some paragraphs explaining how I installed the framework and built the page.
  • Simple text and buttons using Materialize’s pre-designed styles.

Materialize comes with ready-made buttons, text fields, and grids that helped me build a simple but good-looking page without starting from scratch.

3. Challenges and Solutions

While building the page, I ran into a few problems, but I was able to fix them:

Difficulty 1: Setting Up GitHub Pages

The first problem I faced was setting up GitHub Pages. I wanted to use the GitHub Desktop app to upload my code, but for some reason, the app wouldn’t open. After trying to fix it by restarting my computer and reinstalling the app, I decided to upload my code directly on GitHub’s website instead. That worked perfectly!

Difficulty 2: Understanding Materialize’s Grid System

Another problem was understanding how Materialize’s grid system worked. I wanted my page to look good on both small phones and big screens, but at first, I didn’t know how to use the grid system. After reading the Materialize documentation and trying out different things, I figured it out. I used rows and columns to arrange the content and make it responsive.

Difficulty 3: Styling Issues

The last issue I had was with the layout of my text and buttons. They weren’t aligned the way I wanted. This was because I didn’t fully understand how to use the Materialize classes for spacing and alignment. But after looking at examples and experimenting, I was able to fix it by adding the right classes to control the margins and align everything properly.

Conclusion

In the end, I was able to build a simple, responsive webpage using Materialize. Even though I faced some challenges with setting up GitHub Pages and learning how to use Materialize, it was a great learning experience. I now feel more confident using web frameworks and understanding how to create good-looking websites quickly.
